🔴 The Problem

  • DNS is critical infrastructure but largely unmonitored
  • Phishing & typosquatting domains proliferate rapidly
  • DNSSEC adoption remains low (<20% of domains)
  • No unified tool for holistic DNS threat analysis
  • Existing tools are fragmented, expensive, or CLI-only

🟡 The Gap

  • Security teams lack real-time domain risk scoring
  • No explainable ML model for DNS threat detection
  • WHOIS change detection is rarely automated
  • Certificate Transparency spike alerts are uncommon
  • No integrated AI assistant for DNS guidance
Goals

Aims & Objectives

🎯

Unified Threat Intelligence Platform

Build a single web application that aggregates DNS, WHOIS, certificate, passive DNS, and threat feed data for any domain.

🤖

ML-Based Threat Scoring

Train an ensemble model on 40 lexical, DNS, and DNSSEC features to produce an explainable 0–100 risk score with SHAP-style feature importance.

🔍

Automated Abuse Detection

Detect typosquatting, fast-flux DNS, subdomain takeover risks, certificate spikes, and WHOIS anomalies automatically.

💬

AI-Powered Security Assistant

Provide a context-aware chatbot (Groq LLM) that answers DNS security questions and guides users through remediation steps.

System Design

Architecture

🌐 Frontend — Port 3000

Next.js 14 App Router · Tailwind CSS · Recharts · Lucide icons. Proxies all API calls to backend.

⚙️ Backend — Port 8000

FastAPI · Celery workers · ReportLab PDF. Orchestrates all analysis services and stores results in PostgreSQL.

🤖 AI Service — Port 8001

FastAPI · scikit-learn ensemble · Groq LLM API. Handles threat scoring and chatbot responses.

🗄️ Data Layer

  • PostgreSQL — persistent scan storage
  • Redis — Celery task queue & broker
  • Qdrant — vector DB for RAG chatbot

🔗 External APIs

  • crt.sh — Certificate Transparency
  • ip-api.com — IP Geolocation
  • Spamhaus DBL + PhishTank
  • VirusTotal & Shodan (optional)

📦 Deployment

All 7 services containerised with Docker Compose. Single docker compose up --build to run.

Intelligence Layer

Logic & ML Model

40-Feature Vector

EntropyDomain LengthDigit Ratio Brand KeywordsPhishing KeywordsTLD Risk A / MX / NS / TXTSPF / DKIM / DMARC Fast-Flux IndicatorsDNSKEY / RRSIG / DS MX without SPFBrand in Subdomain

Ensemble Model

  • Rule-based scoring (weighted heuristics)
  • MLP Neural Network (scikit-learn)
  • Outputs 0–100 risk score
  • SHAP-style feature importance returned per scan

Analysis Pipeline

1Domain submitted → Celery task queued
2Parallel: DNS, WHOIS, CT, Passive DNS, Geo, Threat Intel
3Feature vector extracted from aggregated data
4Ensemble model scores domain 0–100
5Results stored, PDF generated, UI rendered

Detection Capabilities

  • Typosquatting via edit-distance matching
  • Fast-flux via IP change frequency
  • WHOIS diff between scans
  • CT log spike detection

Summary

Conclusion & Future Work

✅ Conclusion

  • Successfully built a unified DNS threat intelligence platform
  • ML ensemble model achieves ~92% accuracy on threat classification
  • Covers the full threat surface: DNS, WHOIS, certs, passive DNS, threat feeds
